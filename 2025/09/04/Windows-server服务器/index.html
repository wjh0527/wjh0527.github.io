Windows Server 服务器配置与安全管理笔记

基于Windows Server 2012系统，涵盖文件服务器、Web服务器、FTP服务器、DNS服务器、DHCP服务器的配置与管理，以及Windows系统入侵排查与安全加固核心内容，适用于企业内网服务器部署与运维场景。

一、文件服务器配置与管理

文件服务器用于企业内网文件共享、权限控制及存储空间管理，核心解决大文件传输效率与资源访问安全性问题。

1.1 核心理论基础

1.1.1 文件服务器概念

  • 局域网核心服务之一，提供文件共享、权限保护、大容量存储三大核心能力，可基于普通服务器或专用NAS设备部署。
  • 支持多用户同时访问，通过权限控制实现不同用户对文件的差异化操作（如读取、修改、删除）。

1.1.2 文件共享权限（7种核心权限）

权限类型 核心功能 依赖关系
完全控制 无限制访问，可修改、删除文件/文件夹，自动包含以下5种权限 -
修改 可修改文件内容、删除文件，自动包含“读取和运行”“列出文件夹目录”“读取”“写入” 需包含下属4种权限
读取和运行 可读取文件、运行程序 依赖“列出文件夹目录”“读取”权限
列出文件夹目录 仅可浏览文件夹结构，无法读取文件内容 -
读取 可读取文件数据和属性 -
写入 可修改文件内容和属性 -
特别权限 对上述6种权限的细分（如仅允许修改文件属性，不允许删除） 自定义场景使用

1.1.3 访问用户组分类

用户组 权限级别 适用场景
Administrators 系统最高权限，可完全控制所有文件/文件夹 管理员账户
Power Users 仅次于管理员，可修改系统设置，但无法添加管理员账户 高级运维人员
Users 仅运行已验证应用，可创建本地组但仅修改自身创建的组 普通员工
Guests 等同于Users权限，但限制更多（如无法保存文件到系统盘） 临时访客
Everyone 包含所有登录用户（含 Guests） 公开文件共享（需谨慎使用）
System 系统内置账户，拥有最高权限，不可作为登录账户 系统进程运行

1.2 文件服务器安装（Windows Server 2012）

  1. 打开【服务器管理器】→【添加角色和功能】，选择【基于角色或基于功能的安装】。
  2. 服务器选择：从服务器池选中目标服务器（IP：10.6.64.8/24）。
  3. 勾选【文件和存储服务】→【文件和iSCSI服务】→【文件服务器】+【文件服务器资源管理器】。
  4. 功能默认下一步，点击【安装】，完成后在【服务器管理器】→【文件和存储服务】中管理。

1.3 核心配置：文件共享与权限控制

以“创建C:\share共享文件夹，仅user01用户可访问”为例：

1.3.1 新建共享（SMB共享-高级）

  1. 【服务器管理器】→【文件和存储服务】→【共享】→【新建共享向导】。
  2. 选择【SMB共享-高级】（支持配额、文件分类），路径输入C:\share，共享名称设为“share”。
  3. 其他设置：勾选【启用基于存取的枚举】（仅显示用户有权访问的文件）、【加密数据访问】（传输加密）。
  4. 权限配置：
    • 点击【自定义权限】→【添加】，输入用户“user01”。
    • 为user01勾选【完全控制】权限，应用后完成共享创建。

1.3.2 枚举功能配置（用户隔离访问）

实现“user01仅访问share01，user02仅访问share02”：

  1. 在C:\share下创建share01、share02文件夹。
  2. 右键share01→【属性】→【安全】→【高级】→【禁用继承】→【从此对象中删除所有已继承的权限】。
  3. 【添加】user01，赋予【完全控制】权限；同理为share02添加user02权限。
  4. 客户端验证：登录user01账户，访问\\10.6.64.8\share，仅显示share01。

1.4 安全与资源管控

1.4.1 文件屏蔽（禁止危险文件）

禁止share03文件夹存放.exe文件：

  1. 【服务器管理器】→【工具】→【文件服务器资源管理器】→【文件屏蔽管理】→【创建文件屏蔽】。
  2. 路径设为C:\share\share03，模板选择【阻止可执行文件】，创建后放入.exe文件会提示“访问被拒绝”。

1.4.2 全局配额（限制存储容量）

为share文件夹设置100MB配额：

  1. 右键share共享→【配置配额】→选择【100 MB 限制】模板。
  2. 超过配额时，文件传输失败，服务器记录配额告警日志。

二、Web服务器配置与管理（IIS）

Web服务器基于IIS（Internet Information Services）部署，用于企业内网网站发布、多站点隔离及远程更新，适用于电子商务、内部办公系统场景。

2.1 核心理论

2.1.1 IIS支持服务

  • Web服务器：提供HTTP/HTTPS服务（默认端口80/443）。
  • FTP服务器：用于网站文件远程更新。
  • NNTP服务器：新闻组服务。
  • SMTP服务器：邮件发送服务。

2.1.2 HTTP协议特点

  • 无连接：每次连接仅处理一个请求，响应后断开。
  • 无状态：不记忆历史请求，需通过Cookie/Session维持状态。
  • 灵活：支持任意数据类型（通过Content-Type标记）。

2.2 IIS安装与基础配置

2.2.1 IIS安装步骤

  1. 【服务器管理器】→【添加角色和功能】→勾选【Web服务器（IIS）】，自动添加依赖功能。
  2. 【角色服务】默认勾选【常见HTTP功能】（静态内容、默认文档）、【安全性】（请求筛选）。
  3. 安装完成后，【工具】→【Internet Information Services（IIS）管理器】进入管理界面。

2.2.2 创建基础Web站点

发布“某公司网站”（首页homepage.html，路径C:\某公司网站）：

  1. 停止默认站点（Default Web Site），右键【网站】→【添加网站】。
  2. 网站名称：“某公司网站”，物理路径：C:\某公司网站，IP：10.6.64.8，端口：80。
  3. 【默认文档】→【添加】→输入“homepage.html”，调整优先级至最高。
  4. 客户端访问：浏览器输入http://10.6.64.8，显示首页。

2.3 多站点部署（3种核心方式）

2.3.1 基于IP地址（多IP对应多站点）

  1. 服务器添加多IP：【网络共享中心】→【以太网】→【属性】→【IPv4】→【高级】→【添加】（如10.6.64.9、10.6.64.10）。
  2. 新建站点：分别绑定10.6.64.9、10.6.64.10，物理路径对应不同文件夹。
  3. 访问：http://10.6.64.9访问站点1，http://10.6.64.10访问站点2。

2.3.2 基于主机名（多域名对应单IP）

  1. 配置DNS：在DNS服务器添加域名映射（如www1.wgs.com、www2.wgs.com→10.6.64.8）。
  2. 新建站点：绑定IP 10.6.64.8，主机名分别设为www1.wgs.com、www2.wgs.com。
  3. 客户端访问：输入http://www1.wgs.com访问对应站点。

2.3.3 基于端口（单IP多端口）

  1. 新建站点：IP均为10.6.64.8，端口分别设为8081、8082。
  2. 访问：http://10.6.64.8:8081、http://10.6.64.8:8082。

2.4 虚拟目录与远程更新

2.4.1 虚拟目录（多部门资源隔离）

为“某公司网站”添加财务部（CW）、人事部（RS）虚拟目录：

  1. 右键站点→【添加虚拟目录】，别名为“CW”，物理路径C:\某公司网站\CW。
  2. 客户端访问：http://www.wgs.com/CW访问财务部页面。

2.4.2 FTP远程更新

通过FTP更新网站文件：

  1. 右键站点→【添加FTP发布】，绑定IP 10.6.64.8，端口21，SSL选择【无】。
  2. 身份验证勾选【基本】，授权【所有用户】，权限【读取+写入】。
  3. 客户端用FlashFXP登录ftp://10.6.64.8，上传文件实现网站更新。

三、FTP服务器配置与管理

FTP服务器用于企业跨部门文件传输、分公司资源共享，支持多站点隔离、用户权限控制，核心工具为IIS自带FTP服务与Serv-U。

3.1 核心理论

3.1.1 FTP连接模式

模式 数据连接方向 端口使用
主动模式（PORT） 服务器主动连接客户端随机端口（>1024） 控制端口21，数据端口20
被动模式（PASV） 客户端主动连接服务器随机端口（>1024） 控制端口21，数据端口随机

3.1.2 传输模式

  • ASCII模式：适用于文本文件，自动转换换行符（如Windows→Linux）。
  • 二进制模式：适用于程序、压缩包等，逐位传输，不修改数据。

3.2 IIS FTP站点配置

3.2.1 单FTP站点创建

  1. 【服务器管理器】→【添加角色和功能】→勾选【Web服务器（IIS）】→【FTP服务器】。
  2. 【IIS管理器】→右键【网站】→【添加FTP站点】，名称“FTP站点”，路径C:\FTP。
  3. 绑定IP 10.6.64.8，端口21，SSL【无】；身份验证【基本】，授权【所有用户】，权限【读取+写入】。

3.2.2 多FTP站点部署（基于端口）

  1. 新建FTP站点，端口设为2121，路径C:\FTP\分公司。
  2. 客户端访问：ftp://10.6.64.8:21（主站点）、ftp://10.6.64.8:2121（分公司站点）。

3.3 Serv-U FTP配置（第三方工具）

3.3.1 Serv-U安装

  1. 运行安装包，选择中文，接受协议，安装路径默认C:\Program Files\RhinoSoft.com\Serv-U。
  2. 勾选【创建桌面图标】，完成安装后启动Serv-U管理控制台。

3.3.2 新建域与用户

  1. 【新建域】→名称“wgs.com”，协议选择【FTP】，端口21，IP 10.6.64.8。
  2. 【用户】→【添加】，登录ID“ftpuser”，密码“123456”，根目录D:\ServU。
  3. 【目录访问】→添加路径D:\ServU，赋予【读取+写入】权限。
  4. 客户端验证：访问ftp://10.6.64.8，登录ftpuser即可上传/下载文件。

四、DNS服务器配置与管理

DNS服务器用于域名与IP地址映射，解决“记IP难”问题，支持内网域名解析、分公司域名委派及外网转发。

4.1 核心理论

4.1.1 DNS查询类型

查询类型 方向 场景
正向查询 域名→IP 访问网站（www.wgs.com→IP）
反向查询 IP→域名 日志分析（IP溯源）
递归查询 客户端仅请求一次，DNS服务器全程处理 客户端（如PC）查询
迭代查询 DNS服务器返回下一级DNS地址，客户端继续查询 DNS服务器间查询

4.2 DNS安装与正向查找区域

4.2.1 DNS安装

  1. 【服务器管理器】→【添加角色和功能】→勾选【DNS服务器】，安装完成后【工具】→【DNS】进入管理。

4.2.2 创建正向查找区域（wgs.com）

  1. 右键【正向查找区域】→【新建区域】→选择【主要区域】，区域名称“wgs.com”。
  2. 区域文件默认“wgs.com.dns”，动态更新选择【不允许】，完成创建。
  3. 【新建主机】：名称“www”，IP“10.6.64.8”，创建www.wgs.com映射。

4.3 域名委派与转发器

4.3.1 分公司域名委派（sz.wgs.com）

  1. 深圳分公司DNS服务器（IP：10.6.64.9）创建sz.wgs.com正向区域，添加www.sz.wgs.com→10.6.64.9。
  2. 总公司DNS右键wgs.com→【新建委派】，委派域名“sz”，添加分公司DNS IP 10.6.64.9。
  3. 验证：客户端查询www.sz.wgs.com，自动转发至分公司DNS解析。

4.3.2 转发器（解析外网）

  1. 右键【转发器】→【属性】→【编辑】，添加ISP DNS（如202.96.128.166）。
  2. 客户端查询外网域名（如www.baidu.com），DNS服务器转发至ISP DNS解析。

五、DHCP服务器配置与管理

DHCP服务器自动分配IP地址、网关、DNS等参数，解决手动配置IP冲突、工作量大的问题，支持跨网段中继。

5.1 核心理论

5.1.1 DHCP工作流程（DORA过程）

  1. Discover：客户端发送广播请求IP（源IP：0.0.0.0，目标IP：255.255.255.255）。
  2. Offer：DHCP服务器响应，提供IP、子网掩码、租期。
  3. Request：客户端选择第一个Offer，广播确认。
  4. Ack：服务器确认，客户端获取IP并生效。

5.1.2 核心概念

  • 作用域：IP地址分配范围（如10.6.64.1-10.6.64.254）。
  • 排除范围：不分配的IP（如服务器IP 10.6.64.8）。
  • 保留：固定IP绑定（如董事长电脑MAC→10.6.64.137）。
  • 租约：IP使用期限（默认8天，到期前客户端续租）。

5.2 DHCP安装与作用域配置

5.2.1 DHCP安装

  1. 【服务器管理器】→【添加角色和功能】→勾选【DHCP服务器】，安装完成后【工具】→【DHCP】进入管理。

5.2.2 创建作用域（10.6.64.0/24）

  1. 右键【IPv4】→【新建作用域】，名称“wgs”，IP范围“10.6.64.1-10.6.64.254”，子网掩码“255.255.255.0”。
  2. 排除范围：添加“10.6.64.8”（服务器IP），租约设为8天。
  3. 激活作用域：右键作用域→【激活】，客户端可自动获取IP。

5.3 高级配置：保留与中继

5.3.1 IP地址保留（固定设备IP）

为董事长办公室电脑配置固定IP（10.6.64.137）：

  1. 在客户端执行 ipconfig /all，获取MAC地址（如A4-DB-30-A5-F1-95）。
  2. 【DHCP管理器】→右键目标作用域→【保留】→【新建保留】：
    • 保留名称：董事长
    • IP地址：10.6.64.137
    • MAC地址：A4-DB-30-A5-F1-95（无分隔符，连续输入）
  3. 客户端验证：执行 ipconfig /renew，获取到固定IP 10.6.64.137。

5.3.2 DHCP中继（跨网段分配IP）

解决“DHCP服务器（10.6.64.8）与客户端（192.168.102.0/24）跨网段”问题：

  1. 配置中继服务器（双网卡）：
    • 网卡1（连接10.6.64.0/24）：IP 10.6.64.1/24
    • 网卡2（连接192.168.102.0/24）：IP 192.168.102.1/24
  2. 安装中继服务：
    • 【服务器管理器】→【添加角色和功能】→勾选【远程访问】→【路由】。
  3. 配置中继代理：
    • 【工具】→【路由和远程访问】→右键服务器→【配置并启用路由和远程访问】→选择【自定义配置】→勾选【LAN路由】。
    • 展开【IPv4】→【DHCP中继代理】→右键→【新增接口】，分别添加“网卡1”和“网卡2”。
    • 右键【DHCP中继代理】→【属性】→添加DHCP服务器IP（10.6.64.8）。
  4. 客户端验证：192.168.102.0/24网段客户端执行 ipconfig /renew，成功获取192.168.102.x网段IP。

六、Windows系统入侵排查

针对Windows服务器入侵场景（如暴力破解、漏洞攻击、木马植入），从账号、进程、日志等维度定位攻击源与攻击行为。

6.1 核心排查维度

6.1.1 账号安全排查

  1. 检查可疑账号与隐藏账号：
    • 命令行执行 lusrmgr.msc，查看【本地用户和组】，重点排查Administrators组新增账号。
    • 注册表排查隐藏账号：regedit→HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account\Users\Names，对比本地用户列表，删除未知账号。
    • 工具辅助：使用D盾_web查杀工具的【克隆账号检测】功能，识别注册表克隆账号。
  2. 登录日志分析：
    • 【事件查看器】→【Windows日志】→【安全】，筛选事件ID：
      • 4624：登录成功（重点关注远程登录IP、时间）
      • 4625：登录失败（排查暴力破解，如短时间内大量失败记录）
    • 使用Log Parser分析日志，命令示例：
      LogParser.exe -i:EVT "SELECT TimeGenerated as LoginTime, EXTRACT_TOKEN(Strings,5,'|') as Username FROM C:\Security.evtx WHERE EventID=4624"

6.1.2 进程与端口排查

  1. 查看异常进程：
    • 命令行：tasklist 查看进程列表，结合 netstat -ano 定位进程对应的端口（如PID对应的可疑连接）。
    • 工具辅助：Process Explorer查看进程路径、签名信息，重点排查无签名、无描述的进程。
    • 结束可疑进程：taskkill /pid [PID] -t -f（-t表示终止进程树）。
  2. 检查异常端口：
    • 命令行：netstat -ano | findstr "LISTENING"，排查非业务端口（如445、3389是否被异常占用）。
    • 禁用危险端口：通过Windows防火墙阻断445（永恒之蓝漏洞端口），命令：
      netsh advfirewall firewall add rule name=deny445 dir=in action=block protocol=TCP localport=445

6.1.3 启动项与计划任务排查

  1. 启动项检查：
    • 【运行】→msconfig→【启动】，取消未知启动项（如非系统进程的.exe文件）。
    • 注册表启动项：HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run，删除可疑键值。
  2. 计划任务检查：
    • 【任务计划程序】→【任务计划程序库】，排查未知任务（如隐藏任务、触发条件异常的任务）。
    • 命令行查看：at（旧系统）或 schtasks /query /v（详细信息）。

6.2 典型攻击场景排查

6.2.1 永恒之蓝（MS17-010）漏洞排查

  1. 漏洞扫描：使用Nmap扫描目标IP，命令：
    nmap --script vuln -p 445 10.6.64.8，查看是否存在漏洞提示。
  2. 进程排查：检查是否有ms17-010.exe等异常进程，通过tasklist | findstr "ms17"定位。
  3. 日志分析：查看【安全日志】中4688（进程创建）事件，是否有异常进程启动记录。

6.2.2 木马与后门排查

  1. 文件扫描：使用卡巴斯基KVRT、火绒等工具全盘扫描，重点排查C:\Windows\System32、C:\Users\Public等目录。
  2. 黏滞键后门排查：检查C:\Windows\System32\sethc.exe是否被替换为cmd.exe（连续按5次Shift弹出CMD即为异常），正常大小约272KB，异常则替换回原文件。
  3. 远程桌面后门：查看【系统日志】中3389端口登录记录，排查未知IP的远程登录。

七、Windows主机操作系统加固

基于《Windows主机操作系统加固规范》，从账号、日志、协议、服务等维度提升系统安全性，共26项核心加固项。

7.1 账号与认证加固

7.1.1 账号管理

  1. 按用户类型分配账号：
    • 【计算机管理】→【本地用户和组】，创建管理员、审计、来宾等专用账号，避免账号混淆。
    • 清理无效账号：删除与业务无关的账号（如默认测试账号），锁定长期未使用账号。
  2. 重命名管理员与禁用Guest：
    • 重命名Administrator：右键账号→【重命名】，改为非默认名称（如Admin_WGS）。
    • 禁用Guest：右键Guest→【属性】→勾选【账户已禁用】。

7.1.2 密码策略配置

  1. 【本地安全策略】→【账户策略】→【密码策略】：
    • 密码必须符合复杂性要求：启用（需包含大小写字母、数字、特殊符号）。
    • 最短密码长度：8位。
    • 密码最长期限：90天。
    • 密码历史记录：记住5个（防止重复使用旧密码）。
  2. 账户锁定策略：
    • 账户锁定阈值：6次无效登录（防止暴力破解）。
    • 账户锁定时间：30分钟。
    • 复位账户锁定计数器：30分钟。

7.2 日志与通信加固

7.2.1 审核策略配置

  1. 【本地安全策略】→【本地策略】→【审核策略】：
    • 审核登录事件：成功+失败（记录登录行为）。
    • 审核账户管理：成功+失败（记录账号新增/删除）。
    • 审核特权使用：失败（记录权限滥用）。
    • 审核系统事件：成功+失败（记录系统关键操作）。
  2. 日志大小配置：
    • 【事件查看器】→右键【应用日志/系统日志/安全日志】→【属性】：
      • 日志大小：不小于8192KB。
      • 达到最大尺寸时：按需要改写事件（避免日志溢出）。

7.2.2 通信协议加固

  1. 启用TCP/IP筛选：
    • 【网络连接】→【本地连接】→【属性】→【IPv4】→【高级】→【选项】→【TCP/IP筛选】→【属性】，仅开放业务所需端口（如80、21、3389）。
  2. 启用SYN攻击保护：
    • 注册表配置：regedit→HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters：
      • 新建SynAttackProtect（DWORD），值为2。
      • 新建TcpMaxHalfOpen（DWORD），值为500（SYN_RCVD状态阈值）。

7.3 服务与启动项加固

7.3.1 关闭无用服务

  1. 【服务】→禁用非业务服务，如：
    • Telnet（远程登录服务）
    • Remote Registry（远程注册表访问）
    • SNMP Service（默认社区名public，需修改或禁用）。
  2. 修改SNMP服务密码：
    • 【服务】→【SNMP Service】→【属性】→【安全】，删除默认“public”社区名，新增自定义社区名（如WGS_SNMP），权限设为“只读”。

7.3.2 关闭自动播放

  1. 【组策略】→gpedit.msc→【计算机配置】→【管理模板】→【系统】→【关闭自动播放】，选择“所有驱动器”，启用配置（防止U盘病毒自动运行）。
