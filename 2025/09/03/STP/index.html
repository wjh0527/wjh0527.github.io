一、STP（生成树协议）基础

1.1 核心作用

  • 解决二层环路问题：消除冗余链路带来的广播风暴、多帧复制、MAC地址表震荡（飘移）三大问题。
  • 实现链路备份：在主链路故障时，备份链路可切换，保障网络可靠性。

1.2 二层环路的危害

危害类型 具体表现
广播风暴 广播帧在环路中无限循环，占用大量带宽，导致网络拥堵
多帧复制 终端设备收到重复的数据帧，影响数据处理准确性
MAC地址表震荡 交换机频繁更新MAC地址与端口的映射关系，导致地址表不稳定

1.3 STP工作原理（四步操作）

1.3.1 第一步：选举根桥

  • 选举依据：基于BID（桥ID），BID由“优先级+MAC地址”组成。
    • 优先级数值越小越优先，默认优先级为32768，调整时需为4096的倍数（范围0-61440）。
    • 优先级相同时，MAC地址数值小的交换机为根桥。
  • 关键命令：
    • 查看BID：dis stp
    • 调整优先级：stp priority 4096（例：将优先级设为4096）
    • 配置根桥：stp root primary（优先级自动设为0，直接成为根桥）
    • 配置备份根桥：stp root secondary（优先级设为8192，根桥故障时补位）

1.3.2 第二步：选举根端口（RP）

  • 定义：非根交换机上“到根桥路径开销最小”的端口，负责发送BPDU。
  • 选举依据（优先级从高到低）：
    1. 根路径开销（到根桥的累计路径成本，数值越小越优）
    2. 对端交换机的BID（数值越小越优）
    3. 对端端口的PID（端口ID，由“端口优先级+端口号”组成，数值越小越优）
    4. 本端端口的PID（数值越小越优）
  • 关键参数：
    • PID默认端口优先级为128，调整时需为16的倍数（范围0-240）。
    • 修改PID命令：[Huawei-Ethernet0/0/1]stp port priority 112（例：将端口0/0/1的优先级设为112）
  • 查看路径开销：在非根桥上执行display stp brief

1.3.3 第三步：选举指定端口（DP）

  • 定义：每个网段上“负责转发数据”的端口，根桥上所有端口默认是指定端口。
  • 选举依据（非根交换机网段）：
    1. 根路径开销（数值越小越优）
    2. 本交换机的BID（数值越小越优）
    3. 本端口的PID（数值越小越优）

1.3.4 第四步：阻塞非根非指定端口

  • 未被选为根端口或指定端口的端口，将进入Discarding（阻塞）状态，仅接收BPDU，不转发数据和BPDU，避免环路。

1.4 STP端口状态与角色

1.4.1 端口状态（5种）

状态名称 功能描述 对应角色
Forwarding（转发） 转发用户流量和BPDU，正常工作状态 根端口、指定端口
Learning（学习） 学习MAC地址，不转发用户流量 根端口、指定端口（过渡状态）
Listening（侦听） 转发BPDU，不转发用户流量，判断端口角色 根端口、指定端口（过渡状态）
Blocking（阻塞） 仅接收BPDU，不转发流量和BPDU 预备端口（ALTE）
Disabled（禁用） 不处理BPDU，不转发流量 无有效角色（端口未启用或故障）

1.4.2 端口角色（3种）

  • 根端口（Root Port）：非根交换机到根桥的最优端口，Forwarding状态。
  • 指定端口（Designated Port）：网段内转发数据的端口，Forwarding状态。
  • 预备端口（Alternate Port）：备份根端口的端口，Discarding状态。

二、RSTP（快速生成树协议）

2.1 与STP的核心差异

  • 收敛速度：STP收敛需30-50秒，RSTP通过P/A（Proposal & Agreement）机制，实现秒级收敛。
  • 端口状态简化：将STP的5种状态简化为3种，减少过渡时间。
STP状态 RSTP状态 对应RSTP端口角色
Disabled Discarding Disable
Blocking Discarding Alternate、Backup
Listening Discarding 根端口、指定端口
Learning Learning 根端口、指定端口
Forwarding Forwarding 根端口、指定端口

2.2 RSTP新增端口角色

角色名称 功能描述
Backup端口 指定端口的备份，提供“从根桥到非根桥”的备用链路
Alternate端口 根端口的备份，提供“从指定桥到根桥”的备用路径

2.3 RSTP关键特性

2.3.1 边缘端口（Edge Port）

  • 定义：连接终端设备（如PC）的端口，不参与RSTP运算，不接收/处理配置BPDU。
  • 优势：从Disabled状态直接进入Forwarding状态，无过渡时延，提升终端接入速度。
  • 配置命令：[SWC-GigabitEthernet0/0/3]stp edged-port enable（将端口0/0/3设为边缘端口）
  • 注意：Sx7系列交换机默认所有端口为非边缘端口。

2.3.2 RST BPDU优化

  • 发送机制：非根桥无论是否收到根桥的BPDU，都会按Hello Timer（默认2秒）间隔主动发送RST BPDU，加快拓扑变化感知。
  • Flag字段扩展：使用STP中未启用的中间6位，标识端口角色、状态等信息，如“Proposal（提议）”“Agreement（确认）”“TC（拓扑变化）”位。

2.3.3 拓扑变化处理

  • 交换机检测到拓扑变化后，发送TC置位的BPDU。
  • 其他交换机收到TC BPDU后，清空所有非接收端口的MAC地址表，仅保留接收端口的映射，加快地址表更新。

2.4 RSTP收敛过程（P/A机制）

  1. 初始状态：所有交换机启动RSTP后，均认为自己是根桥，所有端口为指定端口，处于Discarding状态。
  2. 发送Proposal BPDU：交换机互相发送Proposal置位的BPDU，比较BID确定根桥（BID最优的为根桥）。
  3. 同步过程：非根桥收到更优的Proposal BPDU后，停止发送自身BPDU，阻塞所有非边缘端口，完成同步。
  4. 发送Agreement BPDU：同步完成后，非根桥发送Agreement置位的BPDU给根桥，确认收敛。
  5. 端口状态切换：根桥与非根桥的连接端口分别进入Forwarding状态，P/A机制向下游传递，完成全网收敛。

2.5 RSTP保护机制

保护类型 功能作用 配置命令
根保护 确保根桥的指定端口不会因网络问题改变角色，防止非法交换机抢占根桥 [SWA-GigabitEthernet0/0/1]stp root-protection
BPDU保护 边缘端口收到BPDU时立即关闭，避免终端侧引入环路，支持自动/手动恢复 [SWC]stp bpdu-protection
环路保护 根端口长时间收不到上游BPDU时，进入Discarding状态，防止形成环路 [SWC-GigabitEthernet0/0/1]stp loop-protection

2.6 RSTP配置与验证

  • 配置RSTP模式：[SWA]stp mode rstp（交换机所有端口工作在RSTP模式）
  • 验证配置：[SWA]display stp（查看RSTP全局信息，确认Mode为RSTP）
  • 端口状态验证：[SWC]display stp interface GigabitEthernet 0/0/1（查看指定端口的角色、状态、保护类型等）

三、MSTP（多生成树协议）

3.1 核心优势

  • 兼容STP/RSTP，继承RSTP快速收敛特性。
  • 支持基于VLAN划分“实例”（一个实例对应一棵生成树），实现不同VLAN的流量负载均衡（解决STP/RSTP单根桥无法负载均衡的问题）。

3.2 MSTP关键配置（华为设备）

  1. 启用MSTP模式：[Huawei]stp mode mstp
  2. 进入MST域配置视图：[Huawei]stp region-configuration
  3. 配置MST域参数（同一域内参数需一致）：
    • 配置域名：[Huawei-mst-region]region-name gok（例：域名为gok）
    • 配置修订号：[Huawei-mst-region]revision-level 1（修订号需一致，默认0）
  4. 映射VLAN到实例：[Huawei-mst-region]instance 1 vlan 10（将VLAN10映射到实例1）、[Huawei-mst-region]instance 2 vlan 20（将VLAN20映射到实例2）
  5. 激活MST域配置：[Huawei-mst-region]active region-configuration
  6. 配置实例根桥：
    • [LSW1]stp instance 1 root primary（LSW1为实例1的根桥）
    • [LSW1]stp instance 2 root secondary（LSW1为实例2的备份根桥）
    • [LSW3]stp instance 2 root primary（LSW3为实例2的根桥）

四、三种生成树协议对比

协议类型 收敛速度 根桥数量 负载均衡能力 适用场景
STP 慢（30-50秒） 1个 无 简单小型交换网络，对收敛速度要求低
RSTP 快（秒级） 1个 无 中小型交换网络，需快速收敛，无负载均衡需求
MSTP 快（秒级） 多实例多根桥 支持（基于VLAN实例） 中大型复杂网络，需负载均衡和快速收敛

功能需求 命令

设置交换机优先级 stp priority 数值（需为4096的倍数）
配置根桥 stp root primary
配置备份根桥 stp root secondary

STP操作（工作原理）

1）选举一个根桥（负责数据转发）
2）每个非根交换机选举一个根端口（RP,发送BPDU）
3）每个网段选举一个指定端口（DP,转发数据）
4）阻塞非根、非指定端口

第一步：如何选举根桥？dis stp
根据BID：优先级.MAC地址
优先级小的为根，优先级相同，MAC地址小的为根
优先级默认是32768，可以调整，但必须是4096的倍数，范围（0-61440）
如何查看BID：dis stp
如何调整优先级：stp priority 4096

第二步：每个非根交换机选举一个根端口
根端口：到根桥最近的端口
根路径开销、对端交换机的BID、对端PID和本端PID，小的为根端口
PID：优先级+端口号，端口默认的优先级128

    修改PID：[Huawei-Ethernet0/0/1]stp port priority ?
    INTEGER<0-240>  Port priority, in steps of 16
    在非根桥如何查看路径开销
    [Huawei]display stp brief  (在非根桥上看)

根桥上的所有端口都是指定端

    非根交换机在选举指定端口时分别依据根路径开销、自己的BID、PID
    未被选举为根端口或指定端口的端口为预备端口，将会被阻塞
