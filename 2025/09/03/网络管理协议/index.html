网络管理协议与服务配置学习笔记

一、SNMP（简单网络管理协议）

1.1 核心定位与应用场景

SNMP是用于统一管理不同厂商、不同类型网络设备的协议，通过在网络管理系统（NMS）与被管理设备间传输管理信息，解决大规模网络设备管理复杂的问题，适用于企业级网络中路由器、交换机、服务器等设备的集中监控与配置。

1.2 核心架构

组件 功能描述
NMS（网络管理系统） 管理端，负责发送管理指令（如查询设备状态）、接收设备告警（Trap）
Agent（代理进程） 部署在被管理设备上的进程，接收NMS指令，采集设备状态（如接口流量、CPU使用率），并反馈给NMS
MIB（管理信息库） 设备上的数据库，存储被管理对象的变量（如接口IP、路由表条目），NMS通过访问MIB获取设备信息

1.3 SNMP版本对比

版本 安全性 核心特性 应用场景
SNMPv1 低（无加密，仅社区名认证） 支持Get-Request、Get-Next-Request、Set-Request、Trap四种报文 早期简单网络，安全性要求低的场景
SNMPv2c 中（社区名认证，支持数据加密） 新增Get-Bulk-Request（批量获取数据）、Inform-Request（确认型告警） 当前主流版本，中小型企业网络
SNMPv3 高（支持用户认证、数据加密、访问控制） 定义安全框架，防止报文篡改、窃听，支持多用户权限管理 大型企业、金融等对安全性要求高的网络

二、DHCP（动态主机配置协议）

2.1 核心作用与优势

  • 动态分配IP地址：避免手工配置IP导致的地址冲突、工作量大等问题，适用于主机数量多的企业网络。
  • 集中管理网络参数：除IP外，还可分配子网掩码、网关、DNS服务器地址等参数。

2.2 核心报文类型

报文类型 发送方 功能
DHCP DISCOVER 客户端 首次接入网络时，广播寻找DHCP服务器
DHCP OFFER 服务器 响应DISCOVER，携带分配的IP、租期等参数
DHCP REQUEST 客户端 确认接受某服务器的OFFER，或申请续租IP
DHCP ACK 服务器 确认REQUEST，正式分配IP；或确认续租
DHCP NAK 服务器 拒绝REQUEST（如IP已被占用、租期无法续租）
DHCP RELEASE 客户端 主动释放IP（如客户端下线）

2.3 地址池类型

华为ARG3系列路由器支持两种地址池，适配不同网络场景：

地址池类型 适用场景 配置特点
接口地址池 小型网络（如单网段） 基于接口IP自动生成地址池，无需手动指定网段
全局地址池 大型网络（多网段） 手动定义网段、网关、DNS，可跨接口分配

三、FTP（文件传输协议）

3.1 核心作用

实现客户端与服务器间的文件上传（put）、下载（get），适用于网络设备配置备份、日志导出、系统升级（如路由器VRP系统文件传输）。

3.2 核心机制

  • 双连接模型：
    1. 控制连接：TCP 21端口，用于传输命令（如登录、上传/下载指令），贯穿整个会话。
    2. 数据连接：TCP 20端口（主动模式）或随机端口（被动模式），用于传输文件数据。
  • 传输模式：
    • ASCII模式：传输文本文件（如配置文件）。
    • 二进制模式：传输二进制文件（如系统镜像、日志文件），推荐默认使用。

四、Telnet（远程终端协议）

4.1 核心作用

通过TCP协议（默认端口23）实现远程登录网络设备，适用于管理员对多台设备进行集中配置（如远程维护路由器、交换机）。

4.2 认证模式

认证模式 特点 适用场景
Password认证 仅验证密码，无需用户名 简单网络，设备数量少
AAA认证 验证用户名+密码，支持权限分级 复杂网络，多用户、多权限管理

五、NAT（网络地址转换）

5.1 核心作用

解决IPv4地址枯竭问题，实现私网地址与公网地址的转换，让多个私网主机共用一个或多个公网地址访问Internet。

5.2 NAT类型与适用场景

NAT类型 核心原理 适用场景
静态NAT 私网IP与公网IP一对一固定映射 需外网访问内网服务器（如Web服务器）
动态NAT 私网IP从公网地址池动态获取映射 私网主机数量少，公网地址充足的场景
NAPT（端口复用） 多私网IP映射到同一公网IP的不同端口 私网主机多，公网地址少（如家庭、中小企业）
Easy IP 多私网IP映射到网关出接口的公网IP 网关出接口使用动态获取的公网IP（如拨号上网）
NAT服务器 将公网IP+端口映射到内网服务器IP+端口 外网用户访问内网服务器（如HTTP、FTP服务）

六、ACL（访问控制列表）

6.1 核心作用

通过定义规则对数据包进行分类与过滤，实现：

  • 控制网络访问（如拒绝某网段访问外网）。
  • 限制网络流量（如禁止P2P下载）。
  • 配合NAT、QoS等功能（如指定某网段使用特定公网地址池）。

6.2 ACL分类

ACL类型 编号范围 匹配参数 适用场景
基本ACL 2000-2999 仅源IP地址 简单过滤（如拒绝某网段访问）
高级ACL 3000-3999 源IP、目的IP、协议类型、源端口、目的端口 精细过滤（如拒绝某网段访问特定服务器的HTTP服务）
二层ACL 4000-4999 源MAC、目的MAC、以太帧类型 二层网络过滤（如禁止某PC访问交换机）

6.3 核心规则

  • 匹配顺序：按规则编号从小到大匹配，匹配即停止（默认编号步长5，可手动调整）。
  • 默认动作：所有ACL默认拒绝未匹配的数据包（“隐式拒绝”）。

七、AAA（认证、授权、计费）

7.1 核心概念

AAA是网络安全框架，提供三层功能：

  • 认证（Authentication）：验证用户身份（如用户名+密码），确认是否允许访问。
  • 授权（Authorization）：授予用户访问权限（如仅允许Telnet，不允许修改配置）。
  • 计费（Accounting）：记录用户使用网络资源的情况（如登录时长、流量），用于审计或计费。

7.2 实现协议

  • RADIUS：轻量级协议，适用于远程用户认证（如VPN接入），仅支持UDP。
  • HWTACACS：华为增强型协议，支持更精细的授权（如命令级授权），支持TCP，安全性更高。
