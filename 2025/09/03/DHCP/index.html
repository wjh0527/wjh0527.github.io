一、DHCP基础认知

1.1 核心定义与价值

DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）是一种工作在应用层的网络协议，通过自动为客户端分配IP地址、子网掩码、网关、DNS等网络参数，解决手工配置IP带来的地址冲突、工作量大、管理效率低等问题，适用于企业、校园等主机数量多的网络场景。

1.2 关键技术参数

  • 端口号：采用UDP协议，服务器端使用67端口，客户端使用68端口。
  • 核心功能：动态分配IP地址、集中管理网络参数、自动续租/释放IP地址。

二、DHCP工作原理（四步流程）

DHCP客户端（如PC）从获取IP到完成配置，需经过以下四个核心步骤，均基于广播/单播报文交互：

步骤 报文类型 发送方 核心作用
1 DHCP Discover 客户端 客户端首次接入网络时，通过广播报文寻找网络中的DHCP服务器（此时客户端无IP，目标IP为255.255.255.255）
2 DHCP Offer 服务器 DHCP服务器收到Discover后，通过广播/单播报文响应，携带可分配的IP地址、子网掩码、租期、网关、DNS等参数
3 DHCP Request 客户端 客户端选择最优的Offer（如最先收到的），通过广播报文向服务器确认申请该IP，同时告知其他服务器“无需分配”
4 DHCP ACK 服务器 服务器确认客户端的Request，通过广播/单播报文发送最终配置参数，客户端接收后完成IP配置

2.1 补充流程

  • IP续租：当IP租期到达50%时，客户端向原服务器发送单播DHCP Request申请续租，服务器回复DHCP ACK确认续租。
  • IP释放：客户端主动下线时，发送DHCP Release报文告知服务器释放IP；若租期到期未续租，客户端自动停止使用该IP。

华为设备支持两种核心地址池类型，分别适用于不同网络规模，配置流程如下：

3.1 类型1：基于接口的地址池（小型网络）

3.1.1 适用场景

3.2.1 适用场景

多网段、中大型网络（如企业多部门、多楼层），可手动创建多个地址池对应不同网段，灵活管理。

四、DHCP中继配置（跨网段分配）

4.1 应用场景

当DHCP服务器与客户端不在同一网段时（如服务器在12.1.1.0/24，客户端在192.168.3.0/24），需在中间路由器（中继设备）配置DHCP中继，转发客户端与服务器的DHCP报文，实现跨网段IP分配。

4.2 配置前提

需先配置路由协议（如静态路由、OSPF），确保DHCP服务器、中继设备、客户端所在网段互通。

五、DHCP Snooping配置（防欺骗）

5.1 核心作用

DHCP Snooping是部署在交换机上的安全功能，用于识别合法DHCP服务器，防止非法DHCP服务器（如恶意PC）向客户端分配错误IP，导致网络故障。其核心逻辑是：

  • 默认所有交换机端口为“非信任端口”：仅转发客户端的DHCP Discover请求报文，不转发服务器的DHCP Offer等响应报文。
  • 手动将连接合法DHCP服务器的端口设为“信任端口”：允许转发所有DHCP报文（包括Offer、ACK）。

六、关键命令汇总

功能需求 命令 备注
全局开启DHCP服务 dhcp enable 所有DHCP配置的前提
接口启用接口地址池 dhcp select interface 小型单网段场景
接口启用全局地址池 dhcp select global 中大型多网段场景
创建全局地址池 ip pool 地址池名称 需后续配置网段、网关等参数
配置地址池网段 network 网段 mask 子网掩码 全局地址池必配
排除不分配的IP excluded-ip-address 起始IP 结束IP 避免分配网关、服务器IP
配置DNS服务器 dns-list DNS1 DNS2 常用公共DNS：114.114.114.114
配置IP租期 lease day 天数 hour 小时 minute 分钟 默认1天，可按需调整
启用DHCP中继 dhcp select relay 中继设备的客户端侧接口配置
指定DHCP服务器IP dhcp relay server-ip 服务器IP DHCP中继必配
全局开启DHCP Snooping dhcp snooping enable 交换机安全配置前提
端口标记为信任端口 dhcp snooping trusted 仅连接合法DHCP服务器的端口配置
查看DHCP Snooping配置 display dhcp snooping configuration 验证全局与端口配置

操作配置

DHCP：动态主机配置协议，为客户端自动分配IP地址
使用的端口号：服务端67/udp 客户端68/udp
工作过程：客户端发送DHCP Discover 寻找网络中的DHCP服务器
服务器发送DHCP Offer 告诉客户端可以分配的IP地址
客户端发送DHCP Request 向服务器申请IP地址
服务端发送DHCP ACK 确认分配的Ip地址信息
