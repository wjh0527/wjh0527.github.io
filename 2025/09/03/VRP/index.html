VRP（Versatile Routing Platform）技术文档

一、VRP概述与核心价值

（一）VRP定义与定位

VRP（通用路由平台）是华为数据通信产品的专用操作系统平台，以IP业务为核心，采用组件化体系结构，覆盖路由器、交换机、无线AP等多类网络设备。其核心价值在于：

  • 功能可裁剪与扩展：支持根据设备硬件规格（如中低端交换机、高端核心路由器）和业务需求，灵活启用路由、交换、安全、网管、无线等组件，平衡功能与性能。
  • 提升设备运行效率：通过优化的资源调度机制，解决多终端接入场景下网络设备的负载压力，保障企业网络的稳定运行。
  • 统一管理接口：为不同类型的华为设备提供一致的命令行操作界面和配置逻辑，降低网络工程师的学习成本与运维复杂度。

（二）VRP发展历程

VRP历经多代演进，从早期的集中式设计逐步升级为支持多核、多框的分布式架构，具体发展阶段如下：

发展阶段 时间范围 核心特点 适用场景
VRP1 1998-2001 集中式设计，功能简单 中低端设备（如早期小型路由器）
VRP2 1999-2000 初步分布式平台，支持基础路由特性 中小型企业园区网路由器
VRP3 2000-2004 组件化设计，支持更多业务特性（如VPN、QoS） 中高端路由器与交换机
VRP5 2004-至今 高可靠性架构，支持热备份、故障快速切换 核心路由器、数据中心交换机
VRP8 2009-至今 多进程、多CPU、多框协同，高性能与高扩展性 大型骨干网路由器、云数据中心设备

（三）VRP核心应用场景

VRP通过组件化设计，适配不同网络场景的需求，主要应用方向包括：

  • 路由场景：支持静态路由、动态路由协议（OSPF、BGP、RIP），实现跨网段、跨区域的IP报文转发。
  • 交换场景：支持VLAN划分、Trunk链路、端口安全（Port-Security）、链路聚合（LACP），满足局域网二层通信需求。
  • 安全场景：集成防火墙、ACL（访问控制列表）、ARP欺骗防御、IPsec VPN等功能，保障网络边界与数据传输安全。
  • 无线场景：支持无线控制器（AC）与瘦AP（FIT AP）的协同管理，实现Wi-Fi接入、漫游、流量控制。
  • 网管场景：提供SNMP（简单网络管理协议）接口，支持通过华为eSight等网管平台进行设备监控、配置下发与故障告警。

二、VRP设备登录与初始化

（一）设备管理接口类型

华为网络设备（如AR系列路由器、S系列交换机）提供两种主要的本地登录接口，用于初始配置与故障排查：

接口类型 设备支持 连接方式 核心用途
Console口 所有华为数通设备（如AR2200E路由器、S5720交换机） Console线（一端为RJ45，另一端为DB9或USB转DB9）连接主机COM口 本地初始化配置（如首次登录设置密码、IP地址）
Mini USB口 部分中高端设备（如AR2200E、S6720交换机） Mini USB线连接主机USB口 本地登录备份，无需COM口，兼容性更强

接口实物示例：

  • AR2200E路由器：同时提供Console口与Mini USB口，位于设备前面板，标识分别为“Console”和“Mini USB”。
  • S5720交换机：仅提供Console口，位于设备后面板，标识为“Console”。

（二）Console口登录步骤

1. 硬件连接

  1. 将Console线的RJ45端插入设备的Console口，DB9端（或USB转DB9适配器）插入主机的COM口（或USB口）。
  2. 确保设备通电，主机与设备之间无物理链路中断。

2. 终端软件配置（以SecureCRT为例）

  1. 打开SecureCRT，新建“Serial”连接，选择对应的COM口（如COM3）。
  2. 配置串口参数，需与设备默认Console口参数一致：
    • 波特率（位/秒）：9600
    • 数据位：8
    • 奇偶校验：无
    • 停止位：1
    • 数据流控制：无
  3. 点击“连接”，打开终端窗口。

3. 设备登录与初始化

  1. 重启设备，终端窗口将显示设备启动日志（如BIOS初始化、VRP系统加载）。
  2. 启动完成后，提示“Login authentication”，输入默认用户名（如“admin”）和密码（如“admin@huawei.com”，部分设备首次登录需修改密码）。
  3. 登录后若提示“Auto-Config is working”，输入“y”停止自动配置（避免自动配置覆盖手动设置）：

（三）Mini USB口登录步骤

1. 驱动安装

  1. 将Mini USB线连接设备与主机，主机将识别到新硬件（如“TUSB3410”）。
  2. 安装华为设备配套的Mini USB驱动（如“TUSB3410 Single Driver Wrapper”），按安装向导完成操作（默认安装路径即可）。

2. 终端配置与登录

  1. 打开设备管理器，查看Mini USB对应的COM口（如COM4）。
  2. 终端软件配置与Console口一致（波特率9600等），连接后登录流程与Console口相同。

三、VRP命令行基础

（一）命令行视图体系

VRP采用“视图分层”设计，不同视图对应不同的配置权限与功能范围，核心视图及切换方式如下：

视图名称 标识 核心功能 切换命令
用户视图 <设备名> 查看设备运行状态（如版本、接口状态），无配置权限 设备登录后默认进入
系统视图 [设备名] 配置设备系统参数（如设备名、时区、用户界面） 用户视图下输入“system-view”（可缩写为“sys”）
接口视图 [设备名-接口类型编号] 配置接口参数（如IP地址、端口类型、VLAN） 系统视图下输入“interface 接口名”（如“interface GigabitEthernet 0/0/0”）
协议视图 [设备名-协议名] 配置路由协议（如OSPF、BGP）、安全协议（如ACL） 系统视图下输入协议命令（如“ospf 1”“acl number 2000”）
用户界面视图 [设备名-ui-类型编号] 配置Console/VTY登录参数（如超时时间、密码） 系统视图下输入“user-interface console 0”（Console口）或“user-interface vty 0 4”（VTY远程登录）

（二）命令行操作技巧

1. 快捷键功能

VRP命令行支持多种快捷键，提升操作效率，常用快捷键如下：

快捷键 功能描述 示例
Ctrl+A 光标跳转至当前命令行开头 输入“dis play”，按Ctrl+A后光标到“d”前
Ctrl+C 停止当前命令运行（如中断Ping测试） 执行“ping 192.168.1.1”时，按Ctrl+C终止
Ctrl+Z 从任意视图返回用户视图 在接口视图下按Ctrl+Z，直接回到“”
Ctrl+] 终止当前远程连接（如VTY登录） VTY连接异常时，按Ctrl+]强制断开
Tab键 补全不完整命令 输入“inter”，按Tab键自动补全为“interface”
Backspace 删除光标左侧字符 输入“displa”，按Backspace删除“a”改为“display”

四、VRP文件系统管理

（一）文件系统核心概念

VRP文件系统用于存储设备的配置文件、VRP系统文件、日志文件等，依赖设备的物理存储介质（如Flash、SD卡），核心文件类型如下：

文件类型 用途 默认存储路径 示例文件名
配置文件 保存设备配置（如接口IP、路由协议） Flash:/ vrpcfg.zip（默认保存配置文件）、huawei.cfg（自定义配置文件）
VRP系统文件 设备运行的VRP操作系统镜像 Flash:/ AR2220E-V200R007C00SPC600.cc（AR2200E路由器系统文件）
日志文件 记录设备运行日志（如接口up/down、登录事件） Flash:/log/ logfile.log

（二）基础文件操作命令

VRP提供类似Linux的文件操作命令，用于查询、创建、删除文件/目录，核心命令如下：

操作类型 命令 功能描述 示例
目录查询 pwd 查看当前工作目录 <Huawei> pwd → 输出“flash:/”
文件列表 dir [路径] 显示指定目录下的文件信息（包括文件名、大小、日期） <Huawei> dir flash:/ → 列出Flash根目录所有文件
目录创建 mkdir [路径/目录名] 在指定路径下创建新目录 <Huawei> mkdir flash:/test → 在Flash下创建“test”目录
文件复制 copy [源路径/文件名] [目标路径/文件名] 复制文件到指定路径 <Huawei> copy flash:/vrpcfg.zip flash:/test/ → 复制配置文件到test目录
文件删除 delete /unreserved [路径/文件名] 永久删除文件（跳过回收站） <Huawei> delete /unreserved flash:/test/huawei.cfg → 永久删除test目录下的huawei.cfg
文件重命名 rename [源路径/文件名] [目标路径/新文件名] 修改文件名 <Huawei> rename flash:/test.cfg flash:/new.cfg → 将test.cfg重命名为new.cfg
文件查看 more [路径/文件名] 查看文本文件内容（如配置文件） <Huawei> more flash:/vrpcfg.zip → 查看默认配置文件内容

（三）配置文件管理

配置文件是VRP的核心文件，分为“当前配置文件（Current-Configuration）”和“保存配置文件（Saved-Configuration）”，两者的关系与操作如下：

配置文件类型 存储位置 特点 操作命令
当前配置文件 RAM（内存） 设备运行时的实时配置，未保存前断电丢失 display current-configuration（查看）
保存配置文件 Flash（闪存） 已保存的配置，设备启动时自动加载 display saved-configuration（查看）、save（保存）

配置向日志主机输出日志信息：
#
sysname Router
#
interface Ethernet1/0/0
ip address 11.1.1.1 255.255.255.0 //配置Router的接口IP地址

info-center channel 6 name loghost1 //配置通道6的名称为loghost1
info-center source IP channel 6 log level warning //配置信息通道6允许输出IP模块的日志信息，允许输出的日志级别最低为warning
info-center loghost source Ethernet1/0/0 //配置发送日志信息的源接口
info-center loghost 10.1.1.1 channel 6 //配置日志信息输出到日志主机
#
ip route-static 10.1.1.1 255.255.255.255 Ethernet1/0/0 11.1.1.2 //配置静态路由，保证Router与日志主机之间路由可达
#
return
