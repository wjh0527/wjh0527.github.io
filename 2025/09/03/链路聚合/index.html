链路聚合

蜡笔小王2025-09-032025-09-03

一、链路聚合基础认知

1.1 定义与核心价值

链路聚合（Eth-Trunk）是将多个物理接口捆绑为一个逻辑接口的技术，核心价值体现在三方面：

  • 提升带宽：无需升级硬件，通过多物理链路叠加实现带宽倍增（如2个千兆接口聚合后，逻辑带宽可达2G）。
  • 提高可靠性：采用备份链路机制，单条物理链路故障时，流量自动切换到其他正常链路，避免断网。
  • 实现负载分担：在多链路间合理分配数据流量，避免单条链路过载，优化网络性能。

1.2 应用场景

主要部署在网络核心节点，用于连接高流量需求的设备，例如：

  • 核心交换机与汇聚交换机之间
  • 交换机与路由器之间
  • 数据中心服务器与接入交换机之间（高并发业务场景）

1.3 关键约束条件

Eth-Trunk链路两端的物理接口必须满足以下一致性要求，否则无法正常聚合：

  • 接口数量：两端参与聚合的物理接口数量需匹配
  • 速率：所有接口速率需一致（如均为100M、1000M）
  • 双工模式：均为全双工（Full Duplex）
  • 流控方式：两端流控配置需相同
  • 接口类型：需为同类型以太网接口（如均为GigabitEthernet接口）

二、链路聚合的两种模式

2.1 手工负载分担模式

2.1.1 核心特点

  • 无协议参与：Eth-Trunk的建立、成员接口加入完全通过手工配置，不依赖LACP协议。
  • 全链路活跃：所有加入聚合组的物理接口均为活动接口，共同参与数据转发，平均分担流量。
  • 故障切换：某条活动链路故障时，剩余链路自动平均分担流量，但切换延迟相对较大。

    • 可查看逻辑带宽（如Current BW: 3G）、各成员接口状态（UP/DOWN）

2.2 LACP模式（链路聚合控制协议模式）

2.2.1 核心特点

  • 协议协商：设备间通过发送LACP报文协商聚合参数（如活动接口数量、优先级），动态确定活动/非活动接口。
  • 链路备份：支持“活动接口+非活动接口”架构，非活动接口作为备份，故障时快速切换（延迟小）。
  • 灵活性高：可通过配置接口优先级、最大活动接口数，自定义链路聚合策略。

2.2.2 关键概念

  • 活动接口：参与数据转发的接口，数量由max active-linknumber配置（默认1，可调整）。
  • 非活动接口：处于备份状态的接口，仅在活动接口故障时补位。
  • 接口优先级：数值越小，接口被选为活动接口的优先级越高（默认32768）。

  • 查看LACP聚合状态：[LSW1]display eth-trunk 1

三、二层与三层链路聚合配置差异

3.1 二层链路聚合

  • 适用场景：连接交换机与交换机、交换机与终端（如服务器），用于二层数据转发。
  • 核心特点：Eth-Trunk接口默认是二层接口，无需配置IP地址，可直接承载VLAN流量。
  • 配置关键：仅需创建Eth-Trunk并加入物理接口（参考“手工负载分担模式”配置）。

3.2 三层链路聚合

  • 适用场景：连接路由器与路由器、路由器与交换机（三层转发场景），用于三层IP数据转发。
  • 核心特点：需将Eth-Trunk接口从二层切换为三层，再配置IP地址。

四、常见问题与解答

4.1 能否将千兆以太口和百兆以太口加入同一个Eth-Trunk？

  • 不能。Eth-Trunk要求两端物理接口速率一致，千兆与百兆接口速率不匹配，加入后会导致聚合组无法正常工作（接口状态异常或流量转发混乱）。

4.2 哪种链路聚合模式支持链路备份？

  • LACP模式。LACP模式通过“活动接口+非活动接口”架构实现备份，非活动接口在活动接口故障时自动切换；手工负载分担模式下所有接口均为活动接口，无专门备份链路，仅能在故障后剩余链路分担流量，无提前备份机制。

4.3 如何调整LACP模式下活动接口的优先级？

  • 通过lacp priority 数值命令（接口视图）调整，数值越小优先级越高（范围0-65535，默认32768）。例如：[LSW1-GigabitEthernet0/0/1]lacp priority 100，使该接口优先成为活动接口。
